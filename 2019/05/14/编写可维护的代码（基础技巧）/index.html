  • 尽量少用全局变量

    Javascript 使用的是函数作用域，在函数内声明的变量，只有在函数内有效，不能在函数外使用。全局变量则是在函数外部声明，在函数内无需声明就可以使用。

    每一个javascript环境都有一个全局对象，在函数外部使用this进行访问。创建的全局变量都归全局对象所有。

    在浏览器中，使用window表示全局变量本身

  • 全局变量的问题

    全局变量存在于同一个全局命名空间内，很有可能发生命名冲突

  • 全局变量的产生

  • 变量释放时的副作用

    隐含全局变量与明确定义的全局变量有细微的不同。

    1. 使用var创建的全局变量（这类变量在函数外部创建），可以使用delete操作符撤销变量

    2. 不使用var创建的隐含全局变量（即使它是在函数内部创建），也可以使用delete操作符撤销变量

    这说明隐含全局变量不是真正的变量。而是全局对象的属性，属性是可以通过delete操作符删除的，但是变量不可以

  • 不要增加内置的原型

    增加构造函数的原型属性是以恶搞增强功能性的强大的方法，但是有时候该方法会过于强大。

    增加内置构造函数（Object(),Array(),Fuction()等）的原型是很有诱惑力的，但是这可能会严重影响可维护性。因为这种做法使得代码更加不可预测。其他开发中在使用你的代码的时候可能期望的内置的Javascript方法的使用不一致，而不是期望有一些你自己添加的方法。

    并且，给原型添加属性在没有使用hasOwnproperpty()时可能会在循环中出现。这会导致一些混乱。

  • 避免使用隐式类型转换

    JavaScript在语句比较时会执行隐式类型转换。例如：fasle == 0和 “ “ == 0等这类语句后面都会返回true。

    为了避免隐式类型转换导致混淆不清， 子啊使用语句比较时使用 === 和 !===操作符来对数值和类型做比较。

  • 避免使用eval()

    eval()可以将任意字符串当作一个javascript代码来执行。当代码预先就编译好了的（不需要动态运行时决定），就没有理由需要使用eval(). 如果需要在动态时运行生成，也应该有更好的方式在实现。

    setIntval(),setTimeout,function()等构造函数来传递参数。很特殊情况下，会导致类似eval()的隐患。

    setIntval可以看这篇文章：https://blog.csdn.net/jxqiaole/article/details/51673765

    new Fuction()构造函数和eval()比较类似。如果一定需要使用eval()，那么可以考虑使用new Fuction()来代替eval()。这样的好处是：new Function()中的代码将在局部函数空间中运行， 因此代码中的任何采用var定义的变量不会自定成为全局变量。另一种使变量不会自定成为全局变量的方法是：将eval()的调用封装到一个即时函数中。

    另外：eval()可以访问和修改它外部作用域的变量
