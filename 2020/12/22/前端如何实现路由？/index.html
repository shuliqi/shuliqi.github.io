vue-router的两种模式区别
2020-12-22|vue-router

vue-router 是vue.js的官方路由管理器，它与vue.js核心深度集成，让我们用vue.js构建单页应用变得更加轻松。我们知道vue-router 有两种路由的模式可以选择，默认情况下是使用hash模式；在vue-router4中呢，通过history字段来选择 ；官方文档： 不同的历史模式。 而vue-router3可通过mode字段来选择；官方文档： HTML5 History 模式。那么这两种模式有什么区别呢？通过vue-router来看前端是如何实现路由的呢？

关于后端路由

路由这个概念最早是后端出现的，早期我们打开一个页面如：https://www.shuliqi.xxx.com/index它大致的流程是：

  • 浏览器发出请求
  • 服务端监听到端口有请求进来，然后解析url地址
  • 根据服务器的路由配置，返回相应的信息（如返回 html 字符串，json数据，图片等）
  • 浏览器根据数据包的 Content-Type来决定如何解析数据

简单的来说路由就是用来跟后端服务器进行交互的一种方式，通过不同的路径，来请求不用的资源，页面。

前端路由

随着ajax的出现，异步数据请求交互运行在不刷新页面的情况下进行。而异步交互体验的更高版本就是 单页应用。单页应用不仅仅是页面交互无刷新的，连页面跳转都是无刷新的，所以为实现单页应用，就有了前端路由。

跟服务端路由一样，前端路由实现起来也是根据不同的url路径解析然后动态的渲染区域的html内容。但是这样就存在一个问题，每当url变化的时候，都会造成页面的刷新。那如何解决呢？那如何解决url 改变但页面 不刷新的问题，就出现两个主流的解决方式。

history 模式

HTML5标准出现后， history出现了两个新的 API。pushState()，replaceState()这个两个可以改变url并且不会发送请求。

  • history.pushState(data, title [, url])：往历史记录堆栈顶部添加一条记录； data会在onpopstate事件触发时作为
    参数传递过去；title为页面标题，当前所有浏览器都会 忽略此参数；url为页面地址，可选，缺省为当前页地址；
  • history.replaceState(data, title [, url]) ：更改当前的历史记录，参数同上；
  • history.state：用于存储以上方法的data数据，不同浏览器的读写权限不一样；
  • window.onpopstate：响应pushState或replaceState的调用；有了这几个新的API，针对支持的浏览器，

这两个 API 都会操作浏览器的历史记录，而不会引起页面的刷新。但是pushState 是会增加一条历史记录， 而 replaceState 会替换当前的 历史记录。

但是会因为没有#，所以当用户刷新页面之类的操作的时候，浏览器就会给服务器发送请求，当服务器匹配不到路由的时候， 页面就会空白，所以这种方式就需要服务器做一些支持。

你要在服务端增加一个覆盖所有情况的候选资源：如果 URL 匹配不到任何静态资源，则应该返回同一个 index.html 页面，这个页面就是你 app 依赖的页面。

给个警告，因为这么做以后，你的服务器就不再返回 404 错误页面，因为对于所有路径都会返回 index.html 文件。为了避免这种情况，你应该在 Vue 应用里面覆盖所有的路由情况，然后在给出一个 404 页面。
